## 一、项目是什么

200MW/400MWh储能电池在中国电力现货市场做套利。电价低时充电（买电），电价高时放电（卖电），赚价差。核心是**日前申报**：每天中午前提交明天96个15分钟时段的充放电计划。

4个省份：山东、山西、广东、甘肃。

### 阶段1：精确预测路线（PatchTST）

最初思路：用Transformer预测未来96个精确电价 → LP求解器算最优充放电。

| 版本 | 方法 | 山东 vs LightGBM |
|------|------|-----------------|
| V2 | 原始Transformer | -14.6% |
| V3b | PatchTST（修复归一化bug） | -12.8% |
| V4 | +天气因果特征(39维) | -11.5% |
| V5 | +Huber loss+Skip connection | -7.9% |
| V5集成 | 0.70×LGBM + 0.30×PatchTST | **+1.2%** |

**结论**：Transformer单独干不过LightGBM，但集成后小幅超越。验证（Rolling Forward 3/4季度赢，De-Oracle优势保持+2.5%）。

## 三、系统架构

### 区间策略（当前最优方法）

```
离线（一次性）：
  历史价格 → KMeans聚类(K=12, 24维) → 12种日模式
  每种模式 → DP求解 → 最优策略表 (模式×时刻×SOC → 功率)

每日申报（日前）：
  今天数据+明天天气 → GBM分类器 → 概率分布 P(模式1)=0.3, P(模式5)=0.6...
  概率加权混合价格曲线 = Σ P(k) × profile(k)
  混合曲线 blend with 日前价格(40:60)
  DP求解 → 96步充放电方案 → 提交
```

### 关键设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 分类 vs 回归 | 分类（6选1） | 比预测96个数字容易得多 |
| 硬分类 vs 软分类 | 软分类（概率） | 不需要猜中具体模式，概率大致对就行 |
| K=6 vs K=12 | K=12 | 更细的模式，更高天花板 |
| 96维 vs 24维聚类 | 24维 | 去除15分钟噪音，保留小时级信息 |
| MSE vs Huber | Huber | 对价格spike更鲁棒 |
| 每日SOC重置 vs 跨日 | 每日重置50% | 跨日SOC实验失败(-43%)，需更精细实现 |
| 全量 vs 滚动窗口 | 全量（5年） | 365天和全量差不多，180天太少 |

## 九、核心思想总结

### 为什么区间策略比精确预测好

精确预测96个电价 → LP优化 是在解一道不必要的难题。电价预测误差40%，LP对输入极敏感。

区间策略把一个难题拆成两个简单题：
1. 明天是什么类型的日子？（6选1分类，60%+准确率就够）
2. 这种日子怎么操作？（查表，数学最优，零误差）

### 为什么不用强化学习

储能套利可以干净地拆成"预测+优化"两半。优化部分有精确数学解（LP/DP），不需要RL的试错学习。RL适合预测和决策无法分离的问题（自动驾驶、围棋），不适合我们的问题。

### 为什么不用Transformer替代LightGBM

电价预测本质是结构化表格问题（特征已知、关系大部分线性），不是序列建模问题。LightGBM天然擅长这类问题。Transformer在语言上的成功不能简单迁移到时间序列预测。但Transformer作为集成的一部分（30%权重）可以贡献独特的时序洞察。

- 人工交易员（阈值策略）：山东~3700万/年
- LightGBM：山东4416万/年
