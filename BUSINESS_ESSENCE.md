# 储能AI交易策略 — 生意本质全解

## 一、这门生意的第一性原理

### 1.1 电力的致命特性

电力是人类工业文明中唯一**不可经济存储**的大宗商品。石油有油罐，天然气有LNG储罐，粮食有仓库——但电力必须在**生产的同一毫秒被消费**。

这个物理约束产生了一个后果：**电力的供给和需求必须在每一个瞬间精确平衡**。一旦失衡，电网频率偏离50Hz，轻则设备损坏，重则大面积停电。

这意味着电力市场的一切制度设计，本质上都在回答一个问题：

> **谁在什么时间发多少电，以什么价格结算，才能让供需秒级平衡且总成本最低？**

### 1.2 储能的本质：给电力造了一个仓库

储能电站（电池）的出现，第一次让电力变成了"可以存起来的商品"。

有了仓库，你就可以做所有大宗商品交易员都在做的事：

- **低买高卖**：凌晨电价低时充电（买入），傍晚电价高时放电（卖出）
- **卖保险**：告诉电网"我随时可充可放"，赚调频/备用的待命费和调用费
- **削峰填谷**：帮助电网消纳多余新能源发电，政策给补贴

**储能交易的本质 = 在时间维度上做电力的套利**，核心变量是**电价的波动性**——波动越大，套利空间越大。

### 1.3 为什么需要AI

问题的本质是一个**序贯决策问题(Sequential Decision Problem)**：

每15分钟，你面对一个选择：充电、放电、还是等待？

- 你的决策影响未来的状态（SOC变了，剩余容量变了）
- 你不知道未来的电价（只能预测）
- 你有物理约束（不能过充过放，效率损耗，电池寿命损耗）
- 你想最大化一天/一月/一年的总收入

这**不是**一个简单的"低买高卖"问题。因为：

| 你以为的储能交易 | 实际的储能交易 |
|---|---|
| 最低点充满，最高点放空 | 你不知道哪个是最低点、最高点 |
| 充一次放一次 | 一天可能需要充放2-4次 |
| 只看价格 | 还要考虑SOC、效率损耗、降解成本、市场规则 |
| 单一市场 | 需要在能量市场、调频市场、备用市场之间动态分配 |
| 各省规则一样 | 山东、广东、甘肃规则完全不同 |

**如果你有上帝视角（知道未来所有电价），线性规划可以算出最优解。但现实中你只有历史和当前信息。**

这就是RL的用武之地：学习一个策略(Policy)，输入当前可观测信息，输出最优动作——不需要预测未来电价，直接端到端地学"在什么状态下该做什么"。

## 二、中国电力市场现状

### 2.1 市场结构：三个市场

```
┌────────────────────────────────────────────────────────┐
│                 中国电力市场                              │
│                                                          │
│  ┌──────────┐   ┌──────────┐   ┌────────────┐          │
│  │ 中长期市场 │   │ 现货市场   │   │ 辅助服务市场 │          │
│  │          │   │          │   │            │          │
│  │ 年/月/周  │   │ 日前+实时 │   │ 调频/调峰/  │          │
│  │ 合同交易  │   │ 竞价出清  │   │ 备用/爬坡   │          │
│  │          │   │          │   │            │          │
│  │ 锁定~80% │   │ 调节~15% │   │ 保障~5%    │          │
│  │ 基础量价  │   │ 偏差量价  │   │ 安全稳定   │          │
│  └──────────┘   └──────────┘   └────────────┘          │
└────────────────────────────────────────────────────────┘
```

**对储能最相关的是后两个：现货市场（赚价差）和辅助服务市场（赚服务费）。**

### 2.2 现货市场的双结算机制

这是中国和ERCOT最大的区别：

```
                 D-1 下午               D日 实时
                   │                      │
    ┌──────────────▼───────────┐  ┌───────▼────────────┐
    │       日前市场(DAM)       │  │    实时市场(RTM)    │
    │                          │  │                     │
    │  发电厂报价报量            │  │  按实际负荷偏差调整   │
    │  调度统一出清              │  │  15分钟粒度结算      │
    │  确定次日每小时出清价      │  │  价格波动更剧烈      │
    │                          │  │                     │
    │  储能：需报日前计划        │  │  储能：偏差要付罚金   │
    └──────────────────────────┘  └─────────────────────┘

    收入 = DAM中标量 × DAM价格 + (实际偏差量 × RTM价格) - 偏差考核罚金
```

**关键含义：** 储能策略必须做**两层决策**——

1. 前一天下午：报日前充放电计划（承诺明天每小时充/放多少）
2. 实时执行：根据实际电价微调，但偏差要被罚

这比ERCOT（只结算实时价）复杂得多，也是我们AI策略的竞争壁垒所在。

### 2.3 辅助服务市场

| 服务类型 | 响应速度 | 储能优势 | 收入模式 |
|---------|---------|---------|---------|
| **调频(AGC)** | 秒级 | 极强（响应速度是火电100倍） | 里程费 + 容量费 |
| **调峰** | 分钟-小时级 | 强 | 调峰电量 × 补偿价 |
| **备用** | 分钟级 | 强 | 待命容量费 + 调用费 |
| **爬坡** | 秒-分钟级 | 极强 | 爬坡容量费 |

**核心认知：顶级策略不是只做一件事，而是在三个市场之间动态分配功率。**

```
某时段: 调频价格 120元/MW → 分配80%功率给调频，20%给能量套利
下时段: 价差拉到 0.6元/度 → 全部转去做能量套利放电
再下一时段: 电价平稳 → 全部做调频赚服务费
```

### 2.4 各省差异（我们有数据的4省）

| 维度 | 山东 | 广东 | 山西 | 甘肃 |
|------|-----|-----|-----|------|
| **峰谷价差** | 最大（0.7-1.0+元/度） | 中等 | 中等 | 大（负电价频现） |
| **负电价** | 偶发 | 少见 | 偶发 | 频繁（新能源弃电） |
| **调频市场** | 成熟 | 最成熟 | 发展中 | 发展中 |
| **新能源占比** | 高（风电+光伏） | 中 | 高（光伏+风电） | 极高（弃风弃光严重） |
| **储能策略侧重** | 能量套利为主 | 调频为主 | 能量+辅助混合 | 消纳+负电价套利 |
| **市场成熟度** | 先进 | 先进 | 跟进 | 跟进 |

**含义：不能用一个模型打全国，必须省级适配——但模型架构可以复用，只换数据和参数。**

### 2.5 行业痛点

1. **大量电站"趴窝"**：央企因政策要求配了大量储能，但不会运营——很多站一天只充放一次，甚至不充不放
2. **规则策略主导**：目前主流是固定时段充放(TOU)或简单阈值策略，人工设定参数，不能适应电价变化
3. **多市场参与缺失**：大部分电站只做能量套利，没有同时参与辅助服务（因为人工决策无法实时切换）
4. **各省规则不同**：运营团队需要为每个省份手动调参，人力成本高

## 三、我们要做的决策是什么

### 3.1 决策问题的精确定义

**每15分钟，AI系统需要输出一个动作：**

```
动作空间（当前版本 — 离散5动作）:
┌────────────┬─────────────┬──────────────────────┐
│ 动作ID     │ 名称         │ 功率                  │
├────────────┼─────────────┼──────────────────────┤
│ 0          │ wait        │ 0 MW（不操作）         │
│ 1          │ slow_charge │ -60 MW（30%功率充电）  │
│ 2          │ fast_charge │ -200 MW（满功率充电）  │
│ 3          │ slow_discharge │ +60 MW（30%功率放电） │
│ 4          │ fast_discharge │ +200 MW（满功率放电） │
└────────────┴─────────────┴──────────────────────┘
```

**未来扩展（中国市场多市场版本）：**

```
动作 = (能量市场功率, 调频分配比例, 备用分配比例)

例如：总功率200MW
→ 100MW做能量套利放电 + 80MW注册调频 + 20MW注册备用
→ 每15分钟重新分配
```

### 3.2 决策的约束条件

| 约束 | 含义 | 参数 |
|------|------|------|
| SOC上限 | 不能过充 | max_soc = 0.95 |
| SOC下限 | 不能过放 | min_soc = 0.05 |
| 充电效率 | 充100度电只有94.87度进电池 | charge_eff = √0.90 |
| 放电效率 | 放100度电只有94.87度到电网 | discharge_eff = √0.90 |
| 降解成本 | 每充放一度电，电池寿命损失 | $2/MWh |
| 功率上限 | 最大充放功率 | 200 MW |
| 日前计划 | 中国市场需要提前报计划 | 偏差有罚金 |
| 循环寿命 | 电池总共能充放~6000次 | 影响长期决策 |

### 3.3 决策的目标函数

```
最大化: Σ(售电收入 - 购电成本 - 降解成本 - 偏差罚金)

其中:
  售电收入 = 放电量(MWh) × 实时电价(元/MWh)
  购电成本 = 充电量(MWh) × 实时电价(元/MWh)
  降解成本 = |充放电量|(MWh) × 降解单价(元/MWh)
  偏差罚金 = |实际执行 - 日前计划| × 偏差考核价格   ← 中国特有
```

## 四、为什么本质上可行

### 4.1 信息不对称 = 利润来源

储能交易赚钱的根本原因：**电价是可预测的，但大部分电站不预测。**

电价的波动不是随机的，它有强规律：

| 可预测信号 | 信息来源 | 对电价的解释力 |
|-----------|---------|--------------|
| 日前出清价(DAM) | 前一天下午公布 | **最强信号** — DAM是RTM的锚 |
| 时间周期 | 时钟 | 强 — 用电高峰/低谷有固定pattern |
| 天气预报 | 公开数据 | 强 — 决定空调负荷和新能源出力 |
| 近期价格趋势 | 历史数据 | 中 — 短期动量/均值回归 |
| 负荷预测 | 电网公开 | 中 — 反映用电需求 |
| 新能源出力预测 | 电网公开 | 中 — 反映供给侧 |

**我们的ERCOT回测已经证明：仅用价格特征（20个衍生指标），BC策略就比最优规则策略多赚12.2%。加入DAM价格后，预期再提升10-20%。**

### 4.2 规则策略的天花板 = AI的机会

当前主流规则策略的局限：

```
规则策略:
  IF price < 24h均价 × 0.7 THEN 充电
  IF price > 24h均价 × 1.3 THEN 放电
  ELSE 等待

问题:
  ❌ 固定阈值不适应市场变化（夏天电价波动大，冬天小）
  ❌ 只看一个特征（price_ma_ratio），忽略时间、趋势、波动率
  ❌ 没有SOC状态感知（快满了还在充，快空了还在放）
  ❌ 不能多市场联合决策
  ❌ 不能适配省级规则差异

AI策略:
  输入20+特征 → MLP网络 → 输出最优动作

优势:
  ✅ 自适应市场波动（特征包含多时间尺度统计量）
  ✅ 综合多维信息（价格+时间+DAM+趋势+波动率+SOC）
  ✅ 隐式学会"什么时候等待更优"
  ✅ 推理<1ms，完全满足实时决策需求
  ✅ 同一架构换数据即可适配不同省份
```

### 4.3 我们的回测证据

```
ERCOT 2021-2026 Walk-Forward回测（200MW/400MWh）:

  策略            年化收入      vs规则策略    vs理论上限(Oracle)
  ─────────────  ──────────   ──────────   ────────────────
  固定时段(TOU)    ~$6M        Baseline      38% of Oracle
  阈值策略        $10.20M      +70% vs TOU   64% of Oracle
  AI策略(BC)      $11.44M      +12% vs阈值   72% of Oracle
  Oracle(上帝)   $15.83M      理论上限       100%

  AI策略在3/3个测试期全部胜出，年化多赚$1.24M
  对200MW/400MWh电站，这相当于每年多赚约800万人民币
```

### 4.4 天花板还很高

当前AI策略只达到Oracle的72%，还有28%的提升空间。提升路径清晰：

1. **加入DAM前瞻特征**（已实现，合法信息）→ 预期+10-15%
2. **DAgger迭代**（减少分布偏移）→ 预期+3-5%
3. **Offline RL（CQL/Decision Transformer）**（突破BC天花板）→ 预期+5-10%
4. **多市场联合优化**（能量+调频+备用）→ 预期+15-30%（新收入来源）

## 五、数据：Input和Output

### 5.1 Input数据（模型需要什么）

| 数据 | 粒度 | 来源 | 作用 |

| **实时电价(RTM)** | 15分钟 | 省级交易中心 | 决策的直接收入信号 |
| **日前电价(DAM)** | 1小时 | 省级交易中心（D-1公布） | 最强预测信号，前瞻合法 |

#### B. 辅助输入 — 基本面数据（提升预测力）

| 数据 | 粒度 | 来源 | 作用 |
|------|------|------|------|
| 系统负荷 | 15分钟/1小时 | 电网调度 | 反映用电需求 |
| 新能源总出力 | 15分钟/1小时 | 电网调度 | 反映供给侧 |
| 风电出力 | 15分钟/1小时 | 电网调度 | 风电波动大→电价波动 |
| 光伏出力 | 15分钟/1小时 | 电网调度 | 白天光伏压低电价 |
| 联络线功率 | 15分钟/1小时 | 电网调度 | 省间送电影响本省电价 |

### 6.2 典型一天的运营流程（现状 vs AI）

**现状：人工 + 固定规则**

```
06:00  运营人员上班，看昨天电价曲线和今天天气预报
07:00  手动设定今天的充放电时段：
       "10:00-14:00 光伏高峰电价低，充电"
       "17:00-21:00 用电高峰电价高，放电"
08:00  把计划报到调度中心
10:00  开始充电（不管实际电价是多少）
14:00  停止充电
17:00  开始放电
21:00  停止放电
22:00  下班

问题：
  ❌ 10:00-14:00中间有个电价冲高到0.8元的时段，但还在傻充
  ❌ 15:00-17:00电价已经很高了，但计划17:00才开始放
  ❌ 13:00出现负电价（-0.1元），应该加快充电但人已经不看了
  ❌ 完全没参与调频市场（不知道怎么参与 / 来不及手动切换）
```

**AI系统：实时决策**

```
每15分钟：
  模型读取当前电价、DAM价格、SOC、时间特征等29个指标
  → MLP推理（<1ms）
  → 输出: fast_charge / slow_discharge / wait / ...
  → 自动执行

00:00-05:00  电价低，持续充电，但在03:00电价短暂反弹时暂停（规则策略不会）
06:00-09:00  电价中等，等待（规则策略可能已经开始放了）
09:30-10:00  电价突然冲高，快速放电赚差价
10:00-13:00  光伏出力电价走低，重新充电
13:00        出现负电价，满功率充电（被付钱充电）
14:00-16:00  电价回升到中等，等待
17:00-20:00  晚高峰电价冲顶，分阶段放电（先慢放试探，确认高价后快放）
20:00-21:00  电价回落，停止放电保留余量
21:00-23:00  如果调频价格高，切换到调频模式赚服务费
23:00-24:00  开始为次日低谷充电做准备

结果: 一天充放3-4次，每次精准择时，同时参与多个市场
```

### 6.3 现有技术方案对比

| 方案 | 代表 | 方法 | 年化收益(200MW站) | 局限 |
|------|------|------|---------|------|
| 不运营 | 65%的央企配储 | 趴窝 | ~0 | 纯成本 |
| 固定时段(TOU) | 大部分运营中的站 | 谷充峰放，人工设定 | ~600万 | 不适应价格变化 |
| 阈值策略 | 少数专业团队 | price/MA比率决策 | ~1000万 | 只看一个指标，固定参数 |
| **AI策略(我们)** | **本项目** | **BC+特征工程** | **~1120万** | 当前只做能量市场 |
| Oracle上限 | 理论值 | 知道未来全部电价 | ~1580万 | 不可实现 |

## 七、商业可行性总结

### 7.1 价值主张

```
对储能电站运营商（如中广核）:

  "你的200MW储能电站用固定时段策略年赚~600万，
```

### 7.2 为什么壁垒可持续

1. **数据飞轮**：接入越多电站 → 越多真实运行数据 → 模型越好 → 更多客户
2. **省级know-how**：每个省的市场规则、电价特征、出清机制都不同，适配经验是壁垒
3. **多市场联合优化**：从能量套利扩展到调频+备用，技术难度更高，收入天花板更高
4. **合规性**：了解各省交易规则和监管要求，这不是纯技术问题

### 7.3 风险和前提

| 风险 | 影响 | 应对 |
|------|------|------|
| 峰谷价差收窄 | 套利空间减少 | 扩展到辅助服务市场（不依赖价差） |
| 政策变化 | 结算规则调整 | 模型可快速重训练（2分钟） |
| 数据获取 | 拿不到实时数据API | 先用公开数据验证，签约后接私有数据 |
| 竞争者 | 其他AI方案 | 先发优势 + 省级适配经验 |
| 回测vs实盘差距 | 实际收益低于回测 | 影子交易验证 + 保守分成模式 |
